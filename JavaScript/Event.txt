### Event

- 프로그램에 의해 감지되고 처리될 수 있는 동작 or 사건
- 키보드나 마우스 동작처럼 브라우저에서 발생하는 사건
- 사용자가 발생시킬 수도 있고 응용프로그램(JavaScript)이 발생시킬 수도 있음
- 마우스, 키보드, 프레임, 폼, 드래그, 클립보드, 프린트, 미디어 등으로 나뉘어짐


## 이벤트 이름
- 이벤트 타입이라고도 부름
- 이벤트의 종류

## 이벤트 속성
- 이벤트 이름에 on이 붙은 형태
- 이벤트 발생 시, 실행할 코드를 나열한 함수를 지정

## 이벤트 리스너
- 일반적인 프로그래밍에서 이벤트 핸들러라고 불리는 것

## 이벤트 모델
- 문서 객체에 이벤트를 연결하는 방법


## 고전 이벤트 모델
- 문서 객체의 이벤트 속성
- 이벤트를 연결하는 방법
- 이벤트 하나에 하나의 이벤트 리스너만 연결 가능
- 이벤트 리스너 제거 시 문서 객체의 이벤트 속성에 null 지정


## 이벤트 객체
- 브라우저 상에서 발생한 이벤트에 관한 정보를 가짐
- 생성한 이벤트 객체의 내용과 처리 방식이 브라우저에 따라 다름
IE, Chrome : window 객체의 event 속성에 객체 전달
Netscape, Mozilla, Firefox : 이벤트 리스너의 매개 변수로 전달
IE Ver 9부터는 두 가지 모두 지원함
* 2가지 이벤트 객체 전달 방식을 모두 지원하면 됨


## 이벤트 속성
- 속성인 동시에 메서드
- 메서드를 호출하는 것처럼 속성을 호출 시, 이벤트 강제 실행
- 브라우저상에서 발생하는 여러 이벤트들에 대해 동일한 리스너를 작성해야 할 경우 활용


## 인라인 이벤트 모델
- HTML 요소에 이벤트 속성을 추가해 처리할 JavaScript 코드를 적어주는 방법
- 처리할 코드가 너무 길면 이해에 문제가 생김
- HTML 요소에 추가된 이벤트 속성에서 실행할 코드를 함수로 작성하고 연결
- 즉, 특정 이벤트 발생 시 처리할 이벤트 리스너를 정의해 HTML 요소의 추가된 이벤트 속성에서 발생한 이벤트를 매개 변수로 이벤트 리스너 호출


## 기본 이벤트 모델
- HTML 요소 중 일부는 내부적으로 이벤트 리스너에 연결
- 특별한 경우 기본 이벤트를 제거해야 함
- 고전 이벤트 모델에서는 해당 문서 객체의 이벤트 속성 중 기본 이벤트와 연결된 이벤트 리스너에서 false
- 인라인 이벤트 모델의 경우 기본 이벤트와 연결된 이벤트 속성에서 함수 형태로 정의한 이벤트 리스너를 return 키워드 다음에 호출하게 함 


## 표준 이벤트 모델
- W3C에서 공식적으로 지정한 DOM Level2 이벤트 모델
- IE 9부터 지원
- 여러 개의 이벤트 리스너 추가
- 이벤트 전달 방식 중 이벤트 캡쳐링, 이벤트 버블링 지원
- 여러 개의 요소 중복 발생 시, 동시에 이벤트 발생

# 이벤트 캡쳐링
- 중복된 요소 중 가장 위에 있는 요소에 이벤트가 처음 주어지고, 그 후 연속해 자식 요소로 전달

# 이벤트 버블링
- 중복된 요소 중 가장 아래에 있는 요소에 이벤트가 처음 주어지고 그 후에 연속해서 부모 요소로 전달
- JavaScript의 이벤트 전달 방식임

# 이벤트 연결
- 하나의 요소에 대한 여러 개의 이벤트 처리
- 하나의 이벤트에 여러 개의 이벤트 리스너 연결
- addEventListener(event, function, useCapture)
event(발생한 이벤트 이름)
function(이벤트 리스너), 
useCapture(선택적 매개변수, true/이벤트 캡쳐링, false/이벤트 버블링, 기본값은 false) 

# 이벤트 제거
- removeEventListener(event,function,useCapture)
- addEventListener() 메서드를 통해 추가된 이벤트 리스너 제거 시 사용
- addEventListener() 메서드와 매개 변수 의미, 사용이 동일함
- this 키워드를 통해 이벤트가 발생한 객체 참조

# IE 이벤트 모델
- 하나의 이벤트에 여러 이벤트 리스너 연결
- DOM2 단계의 이벤트 모델
- IE11 부터는 지원하지 않음
- 이벤트 캡쳐링 미지원
- 점차 이용 가능성이 줄어들을 것
