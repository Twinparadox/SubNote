일반적인 경우 클라이언트에서 웹 브라우저를 통해 웹 서버에 접속하게 되는 경우, 서버와 클라이언트 사이에 항상 연결이 유지되는 것이 아니고, 클라이언트 요청에 따라 연결이 이루어지며, 응답을 받는 형태임.

웹 브라우저로 웹 서버에 방문한다고 가정했을 때, 연결이 이루어져 해당 사이트의 초기 화면을 제공받는다. 이 상황에서 서버와의 연결은 종료됨.

이후, 초기 화면 상의 메뉴를 클릭하거나 등의 명령을 내렸을 경우, 다시 웹 서버와의 녀결이 이루어지고 웹 서버가 응답하면서 연결이 종료되는 식으로, 요청과 응답이 반복됨.

만일, 사용자 구분 없이 불특정 다수에게 정보를 제공하는 홈페이지와 달리 회원가입을 통해 회원에게만 정보를 제공하려고 할 경우, 클라이언트의 요청 메세지 속에 해당 클라이언트를 식별할 정보가 없으면, 매 요청 시 확인하는 방법이 있어야 함

쿠키는 이럴 때, 웹 서버가 클라이언트를 식별하기 위해 클라이언트의 웹 브라우저에 심어 놓는 정보

웹 서버에서는 이런 쿠키 정보를 이용해 다수의 클라이언트를 식별하기 위한 용도나 사용자 세션을 유지하기 위한 용도로 사용함
