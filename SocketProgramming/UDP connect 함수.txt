connect Function

- TCP 통신은 연결 설정 후, 데이터 전송이 이루어짐
- TCP에서 connect 함수는 소켓에 송/수신 호스트들의 IP주소와 포트 번호를 할당하거나, 연결을 요청


UDP는 연결 없이 통신이 가능하다.
- 클라이언트와 서버 간 전담 소켓이 없음
- sendto, recvfrom 함수 호출 시, 자동으로 IP주소와 포트 번호 할당 수행

UDP에서 connect 함수를 이용해 사전에 할당하면, 커널은 소켓과의 연결 상태를 유지한다.
- 장점 : 더 빠른 속도 제공 + TCP 통신의 read, write 함수 사용
- 단점 : 연결이 구성된 후 지정한 주소로만 송수신 가능



int connect(int sockfd, const struct sockaddr* addr, socklen_t addrlen);
성공 : 0, 실패 : -1

- sockfd : 서버 연결 요청을 위한 클라이언트 소켓 디스크립터
- addr : 서버의 주소 정보를 담은 구조체
- addrlen : 구조체의 크기(byte)



UDP의 데이터 전달 방식

- TCP는 보낸 문자열의 구분 없이 2, 3개씩 한꺼번에 Recv에서 수신 버퍼로 처리 가능
- TCP는 데이터의 경계 없이 전송된 데이터는 송신 버퍼에 넣어 순서대로 수신 버퍼에 전달함

- UDP는 recvfrom 함수와 sendto 함수가 1:1 관계
- 선행된 것을 처리 후 진행함
- 네트워크 카드에 의해 수신된 패킷은 링 버퍼에 저장되어 IP주소 등을 비교해 실제 수신 여부가 비교됨
- 목적지 IP주소가 동일할 때, 소켓 수신버퍼를 통해 응용 프로그램에서 이용하게 됨
